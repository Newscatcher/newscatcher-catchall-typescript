## 1.4.0 - 2026-04-13
* The `JobsClient` now exposes a `getUserJobs()` method that retrieves all jobs created by the authenticated user, supporting optional `page` and `page_size` pagination parameters. The method ordering in `JobsClient` has also been updated to reflect a more logical API flow.
* The SDK now exposes two new resource clients: `client.entities` (EntitiesClient) and `client.datasets` (DatasetsClient), along with new `./entities` and `./datasets` package subpath exports. A new `client.monitors.listMonitors()` method is available to retrieve all monitors for the authenticated user. Error handling has been improved — `CatchAllApiTimeoutError` and `CatchAllApiError` now carry the originating `cause` for easier debugging.
* The SDK now includes a `DatasetsClient` (accessible via `client.datasets`) for creating and managing datasets — named collections of entities that can be connected to jobs for Company Search mode. New methods cover the full lifecycle: listing, creating (from entity IDs or CSV upload), retrieving, updating, deleting, managing entity membership, retrieving status history, and appending entities via CSV.
* The SDK now includes two new resource clients: `EntitiesClient` and `DatasetsClient`. `EntitiesClient` provides methods to list, create (individually or in bulk), retrieve, update, and delete company entities. `DatasetsClient` enables organizing entities into named datasets with support for CSV uploads and paginated entity listing.
* The SDK now supports Datasets and Entities APIs, enabling Company Search mode for jobs. Consumers can create and manage datasets of company or person entities via the new `DatasetsClient` and `EntitiesClient`, then pass `connected_dataset_ids` when submitting a job to receive per-entity relevance scores (via `connected_entities`) on each result record. Over 20 new types have been added to support these capabilities, including `DatasetResponse`, `EntityResponse`, `ConnectedEntity`, `CompanyAttributes`, `DatasetStatus`, `EntityStatus`, and `ValidationReport`.
* The SDK now supports file uploads via the new `Uploadable` type, which accepts file-like objects (streams, buffers, blobs), file system paths, or objects with file data and metadata. A new `FormDataWrapper` class is available for building multipart form-data requests. Additionally, `CatchAllApiError` now exposes an optional `cause` property for richer error context.
