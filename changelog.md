## 1.4.0 - 2026-04-13
* The `JobsClient` now exposes a new `getUserJobs()` method for retrieving all jobs created by the authenticated user, with optional `page` and `page_size` pagination parameters. Error handling has also been updated across several methods: `continueJob` now throws `ForbiddenError` and `UnprocessableEntityError` in addition to `BadRequestError`, and `initialize` no longer throws `UnprocessableEntityError`.
* The SDK now exposes two new resource clients: `client.entities` (`EntitiesClient`) and `client.datasets` (`DatasetsClient`), accessible directly from the top-level client. A new `listMonitors()` method has been added to `MonitorsClient`, enabling retrieval of all monitors created by the authenticated user. Error objects now carry a `cause` property, providing better debugging context for timeout and unknown network errors.
* The SDK now includes a new `DatasetsClient` (accessible via `client.datasets`) with full support for dataset management. Consumers can create, update, delete, and query datasets, manage entities within them, upload CSV files to populate or append data, and retrieve dataset status history.
* The SDK now includes two new resource clients: `DatasetsClient` and `EntitiesClient`. `EntitiesClient` provides methods to create, retrieve, update, delete, and batch-create company entities. `DatasetsClient` provides methods to manage named datasets — including creating datasets from CSV uploads, listing and filtering datasets, and adding or removing entities from a dataset.
* The SDK now supports **Datasets** and **Entities** APIs, enabling Company Search mode for jobs. Use the new `DatasetsClient` to create and manage entity datasets (including CSV uploads), and the new `EntitiesClient` to create, update, and delete entities individually or in batch. Jobs submitted with `connected_dataset_ids` in `SubmitRequestDto` will return result records enriched with a `connected_entities` array, where each entry contains a matched entity ID, name, relevance score (`ed_score`), and a short explanation (`relation`).
* The SDK now supports file uploads via the new `Uploadable` type, which accepts streams, buffers, blobs, file paths, and other file-like objects. A new `FormDataWrapper` class is available for building multipart/form-data requests across Node.js and browser runtimes. `CatchAllApiError` now includes an optional `cause` property to expose the underlying error that triggered the exception.
